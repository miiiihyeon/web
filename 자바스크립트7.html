객체와 변경불가성 - 변경불가성은 객체가 생성된 이후 그 생태를 변경할 수 없는 디자인 패턴 - 객체는 참조형태로 전달을 주고 받는다. 객체가 참조를 통해 공유되어 있다면 상태가 언제든지 변경될 수 있기 때문에 문제가 될 가능성도 커지게 된다. 왜냐하면 객체의 참조를 가지고 있는 어떤 장소에서 객체를 변경하면 참조를 공유하는 모든 장소에서 그 영향을 받기 때문이다. 이것이 의도한 동작이 아니라면 참조를 가지고 있는 다른 장소에 변경 사실을 통지하고 대처하는 추가 대응이 필요하다. 1. immutable value(변경불가성) vs mutable value - 자바 스크립트의 원시타입(boolean, number, string, null, undefined, symbol)은 변경 불가능한 값이다. - 객체타입은 변경 가능한 값이다. 즉 새로운 값을 만들 필요 없이 직접 변경이 가능하다. 2. 불변 데이터 패턴 - 의도하지 않은 객체의 변경이 발생하는 원인의 대다수는 레퍼런스를 참조한 다른 객체에서 객체를 변경하기 때문이다. 이를 해결하는 방법은 객체를 불변객체로 만들어 프로퍼티의 변경을 방지하며 객체의 변경이 필요한 경우에는 참조가 아닌 객체의 방어적 복사를 통해 새로운 객체를 생성한 후 변경한다. 2.1 객체의 방어적 복사(Object.assign) -타킷 객체로 소스 객체의 프로퍼티를 복사한다. 이때 소스 객체의 프로퍼티와 동일한 프로퍼티를 가진 타켓 객체의 프로퍼티들은 소스 객체의 프로퍼티로 덮어쓰기 된다. - 객체의 내용은 변경할 수 있다. 2.2 불변객체화를 통한 객체 변경 방지(Object.freeze) -Object.freeze를 사용하여 불변 객체로 만들 수 있지만, 객체 내부의 객체는 변경가능하다. -내부 객체까지 변경 불가능하게 만들려면 deep freeze를 사용 immutable.js 영구 불변 데이터 구조를 제공 $ npm install immutable
