자바스크립트 객체지향 프로그래밍 실세계에 존재하고 인지하고 있는 객체를 소프트웨어의 세계에서 표현하기 위해 객체의 핵심적인 개념 또는 기능만을 추출하는 추상화를 통해 모델링하려는 프로그래밍 패러다임 - 보다 유연하고 유지보수하기 쉬우며 확장성 측면에서 유리 1. 클래스 기반 vs 프로토타입 기반 1.1 클래스 기반 - 클래스로 객체의 자료구조와 기능을 정의하고 생성자를 통해 인스턴스를 생성 - 모든 인스턴스는 클래스에서 정의된 범위 내에서만 작동하며 런타임에 구조 변경 x 1.2 프로토타입 기반 언어 - 자바스크립트는 명령형, 함수형, 프로토타입 기반 객체지향 언어 - 이미 생성된 인스턴스의 자료구조와 기능을 동적으로 변경할 수 있음 2. 생성자 함수와 인스턴스 생성 - 자바스크립트는 생성자 함수와 new연산자를 통해 인스턴스 생성. => 생성자 함수는 클래스이자 생성자 역할을 함. 3. 프로토타입 체인과 메소드의 정의 - 프로토타입체인 : 프로토타입을 통해 직접 객체에 연결 - 더글라스 크락포드가 제안한 프로토타입 객체에 메소드를 추가하는 방식 /** * 모든 생성자 함수의 프로토타입은 Function.prototype이다. 생성자 함수는 Function.prototype.method()에 접근 @method Function.prototype.method @param ({string})(name)-(메소드이름) @param ({function}) (func) - (추가할 메소드 본체) */ 4. 상속 - 자바스크립트는 기본적으로 프로토타입을 통해 상속을 구현하는데 이는 객체가 다른 객체로 직접 상속된다는 의미이다. *클래스 기반 언어의 상속 방식을 흉내내는 것 *프로토 타입으로 상속을 구현하는 것 4.1 의사 클래스 패턴 상속 - 자식 생성자 함수의 prototype 프로퍼티를 부모 생성자 함수의 인스턴스로 교체하여 상속 구현. 부모와 자식 모두 생성자 함수를 정의해야함 . - 문제점 *new 연산자를 통해 인스턴스를 생성 *생성자 링크의 파괴 *객체 리터럴 4.2 프로토타입 패턴 상속 - Object.create 함수를 사용하여 객체에서 다른 객체로 직접 상속을 구현하는 방식 - 는 매개변수에 프로토타입으로 설정하라 객체 또는 인스턴스를 전달하여 이를 상속하고 새로운 객체를 생성. - 객체리터럴 패턴으로 생성한 객체도 프로토타입 패턴 상속 사용 5. 캡슐화와 모듈패턴 - 캡슐화(정보은닉) : 관련있는 멤버 변수와 메소드를 클래스와 같은 하나의 틀 안에 담고 외부에 공개될 필요가 없는 정보는 숨기는 것 - 모듈패턴 : 객체 내의 메소드가 클로저로서 private변수name접근 - private멤버가 객체나 배열일 경우에 반환된 해당 멤버의 변경 가능 - 모듈패턴은 생성자 함수가 아니며 단순히 메소드를 담은 객체를 반환한다. 반환된 객체는 객체 리터럴 방식으로 생성된 객체로 함수의 프로토타입에 접근 x -> 부모객체로서 상속 x (함수에 반환하는 객체에 모든 메소드를 포함) -> 객체를 반환하는 것이 아니라 함수를 반환해야함.
