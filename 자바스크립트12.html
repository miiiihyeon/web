strict mode 1. strict mode란 - 암묵적 전역 변수 : 전역 스코프에도 변수가 존재하지 않을 때 reference error가 뜰 것 같지만 자바스크립트 엔진은 암묵적으로 전역 객체에 프로퍼티x를 동적 생성 -> 식별자x는 전역 변수 - 개발자의 의도와 상관없이 생성된 전역 변수는 오류를 발생시킬 가능성이 크기 때문에 "반드시" var, let, const 키워드를 사용해 변수를 선언하여야 한다!!! - strict mode는 자바스크립트 언어의 문법을 엄격히 적용하여 기존에 무시되던 오류를 발생시킬 가능성이 높거나 자바스크립트 엔진 최적화에 문제를 일으킬 코드에 대해 명시적 에러를 발생 - 린트 도구는 정적 분석 기능을 통해 소스 코드를 실행하기 전에 소스 코드를 스캔하여 문법적 오류와 잠재적 오류를 찾아 리포팅 해주는 도구 2. strict mode 적용 - 함수 몸체의 선두에 'use strict'추가 function foo(){ 'use strict'; x= 10; // ReferenceError : x is not defined } foo(); - stict mode 는 즉시실행함수로 감싼 스크립트 단위로 적용하는 것이 바람직! 3. strict mode가 발생시키는 에러 3-1. 암묵적 전역 변수 - 선언하지 않은 변수를 참조하면 ReferenceError가 발생 3-2. 변수,함수, 매개변수의 삭제 3-3. 매개변수 이름의 중복. - 중복된 함수 파라미터 이름을 사용하면 SyntaxError 발생 3-4. with문 사용 - SyntaxError발생 3-5. 일반함수의 this - strict mode에서 함수를 일반 함수로 호출할 경우 this에서 undefined가 바인딩. -생성자 함수가 아닌 일반 함수 내부에서는 this를 사용할 필요가 없기 때문이다.
