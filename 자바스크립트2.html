6 데이터타입과 변수 -프로그래밍은 변수를 통해 값을 저장하고 참조하며 연산자로 값을 연산, 평가하고 조건문과 반복문에 의한 흐름제어로 데이터의 흐름을 제어하고 함수로 재사용이 가능한 구문의 집합을 만들며 객체, 배열 등으로 자료를 구조화 하는 것 - 변수는 값의 위치를 기억하는 저장소이다. 값의 위치란, 값이 위치하고 있는 메모리 상의 주소이다. 즉, 변수란 값이 위치하고 있는 메모리 주소에 접근하기 위해 사람이 이해할 수 있는 언어로 명명한 식별자이다. 6-1. 데이터 타입 데이터 타입은 프로그래밍 언어에서 사용할 수 있는 데이터의 종류이다. 자바스크립트의 모든 값은 데이터 타입을 갖는다. 원시타입 : boolean, null, undefined, number, string, symbol 객체타입 : object 6-1-1. 원시타입 -원시타입의 값은 변경이 불가능한 값이며 값에 의한 전달(pass-by-value)이다. 6-1-1-1. number - 모든 수를 실수로 처리하며 정수만을 표현하기 위한 특별한 데이터타입은 없다. - infinity : 양의무한대 / -infinity : 음의 무한대 / NaN: 산술 연산 불가(not a number) ex) var pInf = 10/0 //양의 무한대 console.log(pInf); // infinity - 수학적 의미의 실수는 허수가 아닌 유리수와 무리수를 통틀은 말이지만 프로그래밍 언어에서 실수는 일반적으로 소수를 가리킨다. 6-1-1-2. string 문자열타입은 텍스트 데이터를 나타내는데 사용한다. ex) var str = 'string'; str = 'string'; -문자열은 배열처럼 인덱스를 통해 접근할 수 있다. 이와가은 특성을 갖는 데이터를 유사 배열이라 한다. ex) var str ='string'; for(var i = 0; i< str.length;i++){ console.log(str[i]); }1 str[0]="s"; console.log(str); // string - str[0]="s" 처럼 이미 생성된 문자열의 일부 문자를 변경해도 반영되지 않음. (변경 불가능) but, 새로운 문자열을 재할당 하는 것은 가능하다. 기존 문자열을 변경하는 것이 아니라, 새로운 문자열을 새롭게 할당하는 것이기 때문이다. ex) var str="string"; console.log(str); // string str="String" console.log(str); //String str +="test" console.log(str); // String test 6-1-1-3. boolean 불리언은 false, true 두가지만 나타냄 var foo=" false"; var bar="true"; console.log(typeof foo); //boolean console.log(typeof bar); //boolean 프로그램의 흐름을 제어하는 조건문에서 자주 사용. 비어있는 문자열, null, undefined, 숫자 0 은 false로 간주. 6-1-1-4. undefined -undefined가 유일. 선언은 되었지만 값을 할당하지 않은 변수에 접근하거나, 존재하지 않는 객체 프로퍼티에 접근할경우 undefined가 반환 -이는 변수 선언에 의해 확보된 메모리 공간을 처음 할당이 이루어질 때까지 빈 상태로 내버려두지 않고 자바스크립트 엔진이 undefined로 초기화하기 때문. ex) var foo=; console.log(foo); //undefined 변수의 값이 없다는 것을 명시하고 싶은 경우 undefined를 할당하는 것이 아니라 null을 할당 6-1-1-5. null null타입의 값은 null이 유일. null =/ Null,NULL -의도적으로 변수에 값이 없다는 것을 명시할 때 사용. 이는 변수가 기억하는 메모리 어드레스의 참조 정보를 제거하는 것을 의미 -> 자바스크립트 엔진은 누구도 참조하지 않는 메모리 영역에 대해 가비지 콜렉션을 수행할 것. ex) var foo='lee' foo = null; //참조정보가 제거하는 - 함수가 호출되었으나 유효한 값을 반환할 수 없는 경우, 명시적으로 null을 반환. 예를 들어, 조건에 부합하는 html요소를 검색해 반환하는 document.querySelector()는 조건에 부합하는 html요소를 검색할 수 없는 경우, null을 반환한다. ex) var element = document.querySelector('myelem'); console.log(element); // null -> html 문서에 myelem 클래스를 갖는 요소가 없다면 null을 반환 null타입을 확인할 때 typeof연산자를 사용하면 안되고 일치 연산자(===)사용 6-1-1-6. symbol - 이름의 충돌 위험이 없는 유일한 객체의 프로퍼티 키를 만들기 위해 사용한다. 심볼은 symbol 함수를 호출해 생성. 이때 생성된 심볼 값은 다른 심볼 값들과 다르게 유일한 심볼 var key = symbol('key'); console.log(typeof key);//symbol var obj={}; obj[key]='value' console.log(obj[key]); //value 6-1-2. 객체 타입 - 데이터에 관련한 동작을 모두 포함할 수 있는 개념적 존재. -> 이름과 값을 가지는 데이터를 의미하는 프로퍼티와 동작을 의미하는 메소드를 포함할 수 있는 독립적 주체 -참조에 의한 전달 방식(pass-by reference)으로 전달 6-2. 변수 - 변수는 프로그램에서 사용되는 데이터를 일정 기간동안 기억하여 필요한 때에 다시 사용하기 위해 데이터에 고유의 이름인 식별자를 명시한 것. (식별자는 어떤 대상을 유일하게 식별할 수 있는 이름을 말한다. 식별자에는 변수명, 함수명, 프로퍼티명, 클래스명 등이 있다.) - 변수는 var, let, const 키워드를 사용하여 선언하고 연산자를 이용해 값을 할당한다. 그리고 식별자인 변수명을 사용하여 변수에 저장된 값을 참조한다. -변수명을 통해 데이터의 의미를 명확히 할 수 있어 코드의 가독성이 좋아짐. 6-2-1. 변수의 선언 var x=3; var score = 100; 변수명은 식별자로 불리기도 하며 명명규칙이 존재 ( 반드시 영문자, _ , $ 로 시작. 이어지는 문자에는 숫자도 사용가능 / 대소문자 구별) 6-2-1 중복선언 - 변수는 중복 선언이 가능함. 뒤에 선언한게 나옴. but 사용하지 않는 것이 좋음 6-2-2. 동적 타이핑 - 변수의 타입 지정 없이 값이 할당되는 과정에서 값의 타입에 의해 자동으로 타입이 결정될 것이다. 따라서 같은 변수에 여러 타입의 값을 할당할 수 있다. 6-2-3. 변수 호이스팅 console.log(foo) //undefined var foo = 123; console.log(foo); // 123 { var foo = 456; } console.log(foo) ; //456 호이스팅이란 var 선언문이나 function 선언문 등 모든 선언문이 해당 scope의 선두로 옮겨진 것처럼 동작하는 특성이다. 즉, 자바스크립트는 모든 선언문( var, let, const, function, class)이 선언되기 때문에 이전에 참조 가능하다. 1. 선언단계 (변수 객체에 변수를 등록한다. 이 변수 객체는 스코프가 참조하는 대상이 된다.) 2. 초기화 단계 (변수 객체에 등록된 변수를 메모리에 할당한다. 이 단계에서 변수는 undefined로 초기화 된다.) 3. 할당 단게(undefined로 초기화된 변수에 실제값을 할당한다.) - 자바스크립트의 변수는 다른 c-family와 달리 블록레벨스코프를 가지지 않고 함수 레벨 스코프를 갖느다. 단, let const키워드를 사용하면 블록레벨 스코프를 사용할 수 있다. 함수레벨스코프 : 함수 내에서 선언된 변수는 함수 내에서만 유효하며 함수 외부에서는 참조할 수 없다. 즉, 함수 내부에서 선언한 변수는 지역 변수이며 함수 외부에서 선언한 변수는 모두 전역 변수이다. 블록레벨 스코프 : 코드 블록 내에서 선언된 변수는 코드 블록 내에서만 유효하며 코드 블록 외부에서는 참조할 수 없다. 6-2-4. var키워드의 문제점 1. 함수레벨 스코프 ( 전연 변수의 남발, for loop 초기화식에서 사용한 변수를 for loop 외부 또는 전역에서 참조할 수 있다.) 2. var 키워드 생략 허용 (의도하지 않은 변수값 변경) 3. 변수 호이스팅 ( 변수를 선언하기 전에 참조가 가능) - 전역변수는 유효 범위가 넓어서 어디에서 어떻게 사용될지 파악하기 힘들다. 이는 의도치 않은 변수의 변경이 발생할 수 있고, 여러ㅓ 함수와 상호 의존하는 등 부수효과가 있어 복잡성이 증가한다 -> 변수의 유효범위가 좁을수록 좋음 const와 let사용
