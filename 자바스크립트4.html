제어문 - 주어진 조건에 따라 코드 블록을 실행하거나 반복 실행할 때 사용 1. 블록문 - 0개 이상의 문들을 중괄호로 묶은 것으로 코드 블록 또는 블록이라 부른다. 블록문을 하나의 단위로 취급. - 블록문은 세미콜론을 붙이지 않는다. 2. 조건문 - 주어진 조건식의 평가 결과에 따라 코드 블럭의 실행을 결정. 조건식은 불리언 값으로 평가될 수 있는 표현식. - if ... else 문 / switch 문 2-1. if...else 문 - 주어진 조건식의 평가, 즉 논리적 참, 거짓에 따라 실행할 코드 블록을 결정. 만약, 조건식의 평가 결과가 불리언 값이 아니면 불리언 값으로 강제 변환되어 참 거짓을 구별 ex) if (조건식){ }// 조건식이 참이면 이 코드 블록이 실행 else { }// 조건식이 거짓이면 이 코드 블록이 실행 - 조건식을 추가하고 싶을 경우에 else if 문 사용 if (조건식){ }// 조건식이 참이면 이 코드 블록이 실행 else if(조건식2) { }// 조건식2이 참이면 이 코드 블록이 실행 else{ }//조건식 , 조건식2가 거짓이면 이 코드 블록 실행 2.2 switch문 - switch 문의 표현식을 평가하여 그 값과 일치하는 표현식을 갖는 case문으로 실행 순서를 이동 case문은 상황을 의미하는 표현식을 지정하고 콜론으로 마치고 뒤에 실행할 문들을 위치시킨다. switch문의 표현식과 일치하는 표현식을 갖는 case문이 없다면 실행 순서는 default문으로 이동. default옵션으로 사용할 수 있고 사용하지 x도 있음. - 다양한 사오항에 따라 실행할 코드 블록을 결정할 때 사용. ex) var month = 11; var monthName; switch (month){ case 1 : monthName = 'Jenuary'; break; case 2 : monthName = 'Febraury'; break; case 3 : monthName = 'March'; break; case 11 : monthName = 'November'; break; case 12 : monthName = 'December'; break; default : monthName = 'Invalid month'; } console.log(monthName); //November if...else문으로 해결할 수 있다면 if...else문을 사용하는 편이 좋다. 하지만 switch문을 사용했을 때 가독성이 더 좋다면 switch문을 사용하는 편이 좋다. 3. 반복문 - 주어진 조건식의 평가 결과가 참인 경우 코드 블럭을 실행한다. 그 후, 조건식을 다시 검사하여 조건식이 거짓일 때까지 코드 블록을 다시 실행한다. 3-1. for문 -for (초기화식; 조건식; 증감식){ 조건식이 참인 경우 반복 실행될 문 } ex) for ( var i =0; i<2 ; i++){ console.log(i); } > for (;;){} //무한루프 3-2. while 문 - 주어진 조건식의 평가 결과가 참이면 코드 블록을 계속해서 반복 실행하고 평가 결과가 거짓이 되면 실행을 종료한다. 조건식의 평가결과가 불리언 값이 아니면 불리언 값으로 강제 변환되어 논리적 참, 거짓을 구별한다. ex) var count = 0; while (count <3){ console.log(count); count++; } > // 0 1 2 while (true){ } // 평가 결과가 언제나 참이면 무한루프가 된다. 무한루프를 탈출하기 위해서는 코드 블럭 탈출 조건을 if문에 부여하고 break문으로 코드 블럭을 탈출한다. 3-3. do... while문 - do...while문은 코드 블록을 실행하고 조건식을 평가한다. 코드 블록은 무조건 한번 이상 실행 ex) var count = 0; do{ console.log(count); count++; }while(count<3) > // 0 1 2 // count가 3보다 작을 때까지 코드 블록을 계속 반복 실행한다. 4. break문 - break문은 코드 블록을 탈출한다. 레이블 문, 반복문, switch문의 코드 블록 이외에 break를 사용하면 SyntaxError(문법에러) 발생 (레이블 문 : 식별자가 붙은 문 / 프로그램의 실행 순서를 제어하기 위해 사용 ex) foo //foo라는 식별자가 붙은 레이블 블록문: console.log(1); break foo; //레이블 블록문 탈출 ) 5. continue문 - 반복문의 코드 블록 실행을 현 지점에서 중단하고 반복문의 증감식으로 이동.
